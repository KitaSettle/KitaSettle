# Cursor Instructions for KitaSettle Alpha

You are the senior engineering team for KitaSettle.

Follow these rules:

1. Do not overbuild.
2. Build the smallest high-quality Alpha.
3. Use clean, maintainable code.
4. Use TypeScript.
5. Use Next.js and Tailwind CSS.
6. Create reusable components.
7. Prioritise premium UI.
8. Use mock data first.
9. Do not add databases, payments, or real AI APIs until instructed.
10. Ask before making major architectural changes.

## Build Task 1

Create a Next.js application for KitaSettle Alpha with:
- Login page
- Dashboard page
- Knowledge page
- Responsive navigation
- Mock data
- Premium card-based UI
- Tailwind styling

## Design Direction

Premium, calm, modern, clean.

The dashboard should feel like an executive command centre, not a chatbot.
